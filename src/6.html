// Effect에 대해 배워보자!(Effect.js 파일 참조) 1. 용도 - Effect는 함수를 딱 1번만 실행시키기 위해서 쓰인다. 2. React는 한번 데이터가 render되면 렌더가 이뤄진 변수를 갖고 있는 함수(예를 들자면 App Function) 안에 있는 모든 컴포넌트들이 재실행되는데, 이때 함수도 예외가 없다. 3. 만약에 함수가 타 사이트의 api를 사용하는 것이라면 렌더될 때마다 api를 호출할 순 없으므로 딱 1번만 실행해야 하는데 이럴 때 쓰이는 것이 Effect이다. 4. useEffect를 import 해서 사용 하며 useEffect(함수, []); 형식으로 사용된다. 5. 그리고 []는 deps(의존성 배열)이라고 불린다. 안에는 변수명을 넣어주면 해당 변수가 수정될 때마다 함수를 반복해서 실행하라는 의미가 된다. 6. cleanup이라는 기능도 있는데 잘 쓰이진 않지만 useEffect에 들어가는 함수 안에 return을 적어주고 return으로 함수를 적어주면 해당 컴포넌트가 삭제될 때 해당 펑션을 실행한다. 7. 삭제라는 것은 jsx 내에서 선언된 input의 삭제 유무가 아니라 아예 다른 모듈(Test Function)을 사용하다가 해당 모듈이 사라질 때 삭제됐다고 판단한다.
